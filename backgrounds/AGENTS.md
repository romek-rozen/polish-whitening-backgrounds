# AGENTS.md — backgrounds/

Output directory.  Every subdirectory is **one fitted ZCA background**.

This folder is the public-facing artefact of the whole pipeline — it's
what users pull from the repo and feed to `loader.py`.  Treat it as a
contract: don't rename files inside subdirs, don't change the four-file
layout, don't move artefacts between dirs.

## Read order

1. [`README.md`](README.md) — the user-facing description of what's
   here.  Read it first; the rules below assume you already know the
   naming convention and the file layout.
2. The repo-root [`../AGENTS.md`](../AGENTS.md) — what's running,
   what's deliberately empty, the merge / push contract.
3. [`../GOTCHAS.md`](../GOTCHAS.md) — sharp edges, especially §1 on
   granularity matching.

## Hard rules

1. **Never edit `.npy` files by hand.**  They're produced by SVD
   under floating-point convergence — any byte change invalidates
   the diagnostics that travel alongside in `meta.json`.

2. **Never rename a subdir after the fact.**  The directory name is
   the only stable handle users have — `loader.py` (and external
   consumers, including siteFocus pipelines) load by name.  Renames
   silently break downstream code.  Refit under the new name
   instead, leave the old one in place, and update `REGISTRY.md` to
   point at the replacement.

3. **`meta.json` is generated, not edited.**  If a field is wrong,
   fix the generator in [`../scripts/lib/zca.py`](../scripts/lib/zca.py#L_write_meta)
   and re-fit — don't patch the JSON.  The corpus fingerprint
   especially must stay machine-derived.

4. **Don't commit half-fitted backgrounds.**  A subdir without all
   four files (`W_A.npy`, `mu_A.npy`, `eigvals_A.npy`,
   `<name>.meta.json`) is an aborted run.  Delete the dir, re-run
   `scripts/fit_zca.py`, only then commit.

5. **`REGISTRY.md` and `registry.json` are autogenerated** by
   `scripts/index_backgrounds.py`.  Re-run that script after every
   change here; don't edit the registry files by hand.

## Adding a new background

Don't do it in this folder.  Add the model / dim / corpus to
[`../scripts/run_full.sh`](../scripts/run_full.sh) and let the
pipeline write the subdir.  That guarantees the four-file layout,
the meta.json schema, and the registry refresh all line up.

If you're doing a one-off fit (e.g. trying a different MRL dim or a
different `eps`):

```bash
python scripts/fit_zca.py \
    --chunks data/chunks_qwen_qwen3-embedding-4b \
    --name <new-name> \
    --model qwen/qwen3-embedding-4b \
    --truncate-to <dim>            # MRL slice + L2-renorm before fit
python scripts/index_backgrounds.py # regenerate REGISTRY.md
```

…then commit `backgrounds/<new-name>/` + the regenerated
`REGISTRY.md` + `registry.json` as one atomic change.

## Removing a background

Don't.  Backgrounds are versioned by directory name — removing one
silently breaks any pipeline that was loading it.  Mark it
deprecated in `REGISTRY.md` instead, and ship a replacement under a
new name.

The exception: if a fit completed but turned out to be on a wrong
corpus (e.g. corpus fingerprint doesn't match the rest of the
release), delete it before the commit lands.  Once it's in git
history, leave it alone.

## Sanity-checking a freshly fitted background

Before pushing, eyeball `meta.json.diagnostics`:

- `rank_deficient_eigvals` should be **under ~100** for a 2560-dim
  background; lower for smaller dims.  The v2 4B fits run 2–17.
  Higher counts mean the corpus didn't span the embedding space —
  usually a sign you sliced an MRL dim the model wasn't trained
  at, or the corpus is too small / repetitive.
- `top_ev_ratio_pre` should be in the **tens** for Qwen3.  Values
  near 1 mean the model was already isotropic and whitening
  isn't useful; values in the hundreds mean a single direction
  dominates so much you should investigate (often a tokenization
  bug upstream).
- `eigvals_min` should be **positive** (we add `eps=1e-6` to keep
  it bounded away from zero).  Negative entries mean SVD didn't
  converge cleanly — re-run with a larger `eps`.

If anything in `diagnostics` looks wrong, don't ship the fit.
Delete the subdir and refit on a fixed corpus or with a different
`eps`.

## What's deliberately missing here

- Backgrounds for non-Qwen3 models.  The pipeline is generic — add
  the model to
  [`../scripts/lib/tokenizer.py::OPENROUTER_TO_HF_TOKENIZER`](../scripts/lib/tokenizer.py)
  and the embed step picks it up.  We just don't ship them
  pre-built.
- Backgrounds at MRL dims the model wasn't trained at (e.g. 2048
  for 4B, 1536 for 8B).  The MRL ladder we ship matches each
  model's published targets; off-grid dims work but recall is
  worse.
- The paragraph-level v3 backgrounds.  See [`README.md`](README.md)
  for the timing.
